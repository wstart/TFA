# Changelog

本文件记录 TFA 的重要变更。格式参考 [Keep a Changelog](https://keepachangelog.com)，
版本遵循 [语义化版本](https://semver.org)。

## [0.15.2] — 2026-06-26

### 优化
- **正式签名 + Apple 公证**：改用 **Developer ID** 签名(hardened runtime + 安全时间戳 + entitlements)并经 Apple 公证(notarytool + staple),取代此前的 ad-hoc 临时签名。**双击即可打开**(不再需要右键「打开」绕过 Gatekeeper);更关键的是签名稳定后,macOS 的隐私授权(桌面 / 文稿 / 「访问其他 App 数据」)**持久保留、升级不再重置**——根治此前反复弹窗、终端读不到桌面文件夹的问题。打包脚本内置签名(从钥匙串自动探测身份,公开 clone 无证书时退回 ad-hoc),`NOTARIZE=1` 一键公证。

### 修复
- **大幅减少 macOS Sequoia「访问其他 App 数据」弹窗**:会话持久化的 AI 终端检测**不再探测 `~/.claude/projects/` 文件系统**(每小时快照都会触发该提示),改为只看运行的命令名(claude/codex)。

## [0.15.1] — 2026-06-21

### 修复
- **在工具面板(任务 / 隧道 / CLAUDE.md / Skills / 实验室)上点回**当前 session 无法切回终端界面。根因:打开工具面板时 `selectedConnectionID` 故意保留在原终端,再点同一个时其 `didSet` 因"同值"短路、清除工具面板标志的逻辑没执行。改为所有"激活终端"入口(侧栏点选、⌘1–9、⌘⌥←→、⌘] / ⌘⇧]、⌘K / 搜索跳转)统一走 `activateTerminal`,同值时也强制退出工具面板并重新显示终端。

## [0.15.0] — 2026-06-21

### 新增
- **会话持久化（重启 / tmux 关闭后恢复对话）**：把每个本地终端的会话清单 + scrollback 存进 TFA 自有 SQLite（`~/.tfa/sessions.db`），按永久 `@tfa_id` 关联。重启电脑或 `tmux kill-server` 后,启动时把丢失的会话**重建为休眠占位**(同名 → 同 `@tfa_id` → 环境变量/分组/看板指派自动回连,`cd` 回原目录);**AI 终端自动 `claude --continue` 续接对话**(检测到 Claude/Codex 项目目录),普通终端回放历史文本。保存策略:开机基线 + 每小时 + 优雅退出(⌘Q)完整快照。
- **SSH 反向端口转发（`ssh -R`）管理**：侧栏新增「隧道」面板。`ssh -N -R 远程端口:localhost:本地端口 -p 登录端口 账号@IP`——把本机端口反向暴露到远程服务器。多条配置存储(UserDefaults + 密码进钥匙串)、启停开关、**开机自动拉起**、**断线指数退避自动重连**、**GatewayPorts(允许外网访问)开关**、**实时连接日志查看**。复用现成 `askpassEnv` 注入密码,退出时回收子进程。
- **启动进入 macOS 原生全屏**(独立 Space)。
- **终端区右键 = 侧栏会话右键**:同一套会话功能菜单抽成共享组件,折叠侧栏时也可用。
- 恢复 AI 对话时 `claude --continue` 附带 `--dangerously-skip-permissions`。

### 优化
- **全面换肤(按 DESIGN.md)**：暖色「凸版印刷」浅色外壳——象牙白画布、纯白卡片、亚麻细线描边、**零阴影**、近单色墨黑 + 唯一尘蓝点缀;终端正文区保留固定深色「屏幕」。`Theme.swift` 收口为语义 token,改动一处即全局生效。
- **字体改用系统字(SF + 苹方)**:中英混排大小一致(macOS 衬线无中文配套字,会大小不一,故弃用)。
- **状态色统一为语义意图标准**:`neutral / pending / positive(鼠尾草绿)/ active / attention(琥珀)/ error(砖红)`——全局仅三个彩色(良好/等你/错误),其余单色 + 形状区分,连接/隧道/看板各域统一映射。
- **DESIGN.md 重写为 TFA 专属设计系统**(双表面模型、字体取舍、状态色标准、组件清单)。
- **减少 macOS Sequoia「访问其他 App 数据」弹窗**:`tfa-task` skill 改为内容变更才写、AI 检测结果缓存,减少对 `~/.claude` 的读写。

## [0.14.0] — 2026-06-20

### 新增
- **稳定终端身份（`@tfa_id`）**：每个终端创建时获得一个永久 UUID（写入 tmux 会话环境变量），所有持久数据——**环境变量 / 分组 / 看板任务指派**——一律以它为键。**改名 / 重启后数据原地不动**，不再因会话名变化而丢失（彻底根治此前改名导致环境变量、分组、指派失踪的问题）；首次启动自动把旧的「按名字」数据一次性迁移到新键。
- **任务看板 ⌘⇧T + 看板↔终端互跳**：一键在任务看板与当前终端间往返；卡片 / 编辑器新增「前往终端」直接跳到该 agent 的终端盯进度。
- **`⌘]` 升级为「下一个需要你的终端」**：优先跳转**等待回复 / 受阻**的终端（铃铛），其次才是有新输出的；`⌘⇧]` 保留为只跳待处理。
- **关窗驻留菜单栏**：关闭主窗口不再退出——App 收进菜单栏继续跑**任务调度 / 待接管提醒**（tmux 里的 agent 7×24 工作，调度器也随之常驻），点菜单栏图标恢复窗口；⌘Q 仍是真正退出并回收所有 `tmux -CC` 子进程。

### 优化
- **流畅性 / 减少动画**：键入特效（Typing Combo）**默认关闭**（设置里仍可开）；移除多处常驻 / 循环动画——待处理铃铛不再无限摆动、流式指示点不再脉冲、看板「运行中」标签不再每 0.6 秒重绘；只保留「正在产出输出」的均衡器一个有信息量的动效。整体从「到处在动」变为「只有真有事的地方在动」。
- **启动 / 运行性能**：三个独立轮询循环（活动监测 / 看板监视 / 派发队列）合并为**单一心跳**，并在 App 被完全遮挡时跳过 tmux 轮询；首次 attach 的多条查询改为**流水线并发**（首屏更快，ssh 上更明显）；终端回滚历史**封顶 1 万行**，避免长跑会话内存无界增长。
- **侧边栏精简**：每行只展示**当前文件夹 + 状态**，移除 git 分支 / 监听端口 chip 及其后台探测（不展示就不空跑，省资源）。
- **任务调度可测化**：派发队列 / 重试 / 待接管调和逻辑抽出为独立 `DispatchCenter`（配齐单元测试覆盖忙时排队、连接中补发、任务删除清队、远程拒绝等边界）。
- **右键菜单**：环境变量 / 拷贝 · 粘贴环境变量 / 克隆 · 重启 session 恢复为**一级直达**（不再藏进二级子菜单）。
- **文案统一中文**：菜单栏、空状态、重连 / 启动占位页统一为中文。

### 修复
- **远程（ssh）终端不再可派发任务**：远程 agent 够不到本机看板数据库、永远无法回报，过去会让任务静默停在「进行中」；现在派发 / 回复推送对远程终端明确拒绝并在看板留言说明。
- **看板指派不再静默消失**：终端关闭 / 旧的外部 agent 等情况下，指派人改为**降级显示**（标注「终端已关闭 / 已失联」）而非整行消失；「按终端」分组也为失联指派保留泳道，任务不丢。

### 移除
- **分屏 / 多终端同屏**（tmux 原生 pane 布局 + 应用级多会话并排）：体验不达预期，整套移除，回到「一次只显示一个终端」。

## [0.13.0] — 2026-06-09

### 新增
- **任务看板（Kanban）**：侧边栏新增「任务」面板。把每个任务当 issue 指派给一个终端「agent」并**派发**——一行指令发进它的 tmux 会话让 agent 开干；卡片显示该终端的**实时状态**与**执行记录时间线**（步骤 / 提问 / 受阻，最新在上）。
  - **共享 SQLite 看板**（`~/.tfa/board.db`，WAL）：TFA、`tfa-task` CLI、各终端里的 agent 可并发读写，TFA 实时刷新。
  - 配套 **`tfa-task` 命令行 + `tfa-task` Claude skill**（启动自动安装到 `~/.tfa/bin` 与 `~/.claude/skills`）：让终端里的 agent **注册 / 认领 / 逐步回报(step) / 提问(ask) / 受阻(block) / 完成(done)**，把过程和结果写回看板。
  - **按状态 / 按终端**两种分组（终端多时只列有任务的终端 + 列宽自适应 / 横向滚动）；**待接管**高亮（终端响铃 / agent 提问时卡片标黄）；在详情里**回复会推送到终端**；派发时若终端正忙会**排队**、空闲后自动发（避免键入被运行中的程序吃掉）。
- **分屏 / 多终端同屏**：⌘D 或右键「加入分屏」最多 4 个会话并排（按状态/终端切换无关），记忆上次分屏、点回成员即恢复；只有焦点格收键盘输入。
- **「需要关注」检测**：agent 在后台**停下来等你**时（终端响铃 / OSC 9·777·99 通知）侧栏行变琥珀 + 摇铃 + 显示消息 + 系统通知；**⌘⇧]** 跳到下一个待处理终端。
- **侧边栏 git 分支 + 监听端口**：每个已连接的本地会话行显示 `⎇ 分支` 与进程树正在监听的端口（一次 `lsof`+`ps` 全局共享）。
- **可重排的分组（文件夹）**：拖动文件夹头、或右键「上移 / 下移分组」重新排序，按 host 持久化。
- **新建终端可选起始文件夹**：侧栏 `+` 长按「选择文件夹新建…」（`tmux new-session -c`）。

### 优化
- **终端右键「同步当前文件夹」**：tmux 不会在 `cd` 时发通知，缓存的工作目录会滞后——一键重读真实 cwd，刷新标题栏路径 / 侧栏分支 · 端口 / 文件管理器根目录。
- **精简右键菜单**：从 13 个顶层项收成 7 个，不常用的（分组 / 环境变量 / 会话操作）归入子菜单。

## [0.11.0] — 2026-06-03

### 新增
- **实验室「办公室」可视化**：侧栏底部新增「办公室」入口，把每个 tmux 会话渲染成一个**像素小人**
  （像素美术来自 [pixel-agents](https://github.com/pixel-agents-hq/pixel-agents)，MIT，见 `Resources/office/ATTRIBUTION.md`）。

  - **拖小人到别的分组**即可改归属（与侧栏分组同步）；单击选中、双击进入终端、右键菜单（克隆 / 重启 / 关闭）。

## [0.10.1] — 2026-06-02

### 修复
- **文件管理器崩溃**：打开文件管理器时一次性递归构建整棵目录树，遇到深目录 / 符号链接环 / 大目录
  （如 home）会递归爆栈崩溃（`EXC_BAD_ACCESS`）。改为**惰性加载**——只列根目录，展开某文件夹时才扫下一层，
  递归深度只由用户展开层数决定，物理上不会爆栈，大目录也秒开。Skills 的目录递归顺带加了深度上限防御。

### 优化
- **SSH 密码存储（减少钥匙串反复授权）**：ad-hoc 签名每次重打包 / 重装都变，钥匙串按签名授权，
  导致升级后逐个主机反复弹「允许访问」。现在：
  - 所有主机密码**合并成单条**钥匙串项 → 重签 / 重装最多弹**一次**（而非每主机一次）；
  - 增加**机器绑定的加密备份**（AES-GCM，密钥派生自本机硬件 UUID，不依赖签名）；
  - 钥匙串读取失败时**弹窗询问是否从备份恢复**，恢复后写回钥匙串；A（钥匙串）与 C（备份）始终同步；
  - 旧版逐主机的钥匙串项首次启动**自动迁移**为单条并清理。

## [0.10.0] — 2026-06-02

### 新增
- **文件管理器**：右键本地会话 →「文件管理器…」，以该会话的**当前工作目录**（`pane_current_path`）为根
  打开一个**独立窗口**。
  - 左侧自绘目录树（文件夹优先、隐藏 `.` 开头与 `.bak`、按扩展名图标）；右侧复用 `CodeEditor`：
    **md / py** 语法高亮，shell / json 亦高亮，其余按纯文本编辑；⌘S 保存、首次保存留 `.bak`。
  - 二进制 / 超大文件（>4MB）自动识别为「无法编辑」，不再乱码。
  - 基本管理：新建文件、在 Finder 打开 / 显示、移到废纸篓（可恢复）、刷新；同一路径复用窗口、不同路径各开一个。
  - 远程会话暂不支持（路径在远端机器，本地无法访问），菜单项对远程会话禁用。

## [0.9.0] — 2026-06-01

### 新增
- **重启 session（重载环境变量）**：右键会话即可重启其 shell 并重新应用已保存的环境变量（带确认；会话保留，
  仅中断当前窗格里正在运行的程序）。
- **拷贝 / 粘贴环境变量**：右键「拷贝环境变量」把会话的 env 以 `KEY=VALUE` 写入系统剪贴板；「粘贴环境变量」
  把剪贴板里的 `KEY=VALUE` 合并进目标会话（容忍 `export ` 前缀、`#` 注释、引号），可在会话间或从 `.env` 复制。
- **实验室「PTY / 进程」监控**：显示 pty 用量 / 上限（超 80% 告警）、TFA 的 `tmux -CC` 客户端数、孤儿数、
  残留 tmux 探针 socket 数；一键「清理孤儿客户端」释放 pty（只分离、不结束会话）。

### 修复
- **pty 泄漏导致终端打不开**：TFA 每个终端背后是一个 `tmux -CC` 子进程（各占一个 pty）。被信号杀死
  （`pkill` / 崩溃）时不走正常退出流程，子进程会成为孤儿（PPID=1）继续占用 pty，反复重启后累积到系统
  上限。现在捕获 `SIGTERM` / `SIGINT`，信号退出也会回收所有子进程。

## [0.8.0] — 2026-06-01

### 新增
- **克隆 session**：侧栏右键 →「克隆 session」，按同一套配置（endpoint / host+args / socket / tmux 路径 /
  密码 / 已存的环境变量）开一个**全新会话**，仅会话名是新的。
- **顶部显示当前路径**：主窗口 header 新增 📁 路径胶囊，显示活动终端的当前工作目录（`pane_current_path`），
  过长中间截断、hover 看全；本地路径缩写为 `~`，远程保留原样。

### 修复
- **环境变量改名后丢失**：env / 分组用会话名当 key，但**改名时没同步更新**——重启后会话按新名恢复，
  旧 key 下的 env / 分组就读不到了。现在改名会**同步会话名并迁移持久化数据**，env / 分组跟着走、重启仍在。
- **切到工具面板时终端仍有焦点**：切换实验室 / Skills / CLAUDE.md 时主动交还键盘焦点，隐藏的终端不再
  保留聚焦光标、也不会吞按键。

## [0.7.0] — 2026-05-31

### 新增
- **CLAUDE.md 全局规则编辑器**：侧栏底部新增 `CLAUDE.md` 入口（在 Skills 之上），直接编辑
  `~/.claude/CLAUDE.md`，Markdown 语法高亮、⌘S 保存、首次保存留 `.bak`、文件不存在则保存时自动创建；
  带 **reload 按钮**从磁盘重新加载（有未保存改动会二次确认）。
- **Skills 改为目录树**：每个 skill 作为可展开文件夹浏览其全部文件/子目录；编辑器对 **md / py / shell /
  json** 做语法高亮。

### 修复
- **无法连接 Homebrew 远程 macOS 主机**：ssh 一次性命令走 `shell -c` 不读 `.zprofile`，导致
  `/opt/homebrew/bin` 不在 PATH、裸 `tmux` 找不到而连接失败。现在远程 tmux 调用（含会话发现）统一用
  `env PATH=…` 显式带上 Homebrew / 系统 bin 目录。
- **Skills 目录展不开**：符号链接的 skill（如 gstack 系列）`contentsOfDirectory` 返回空，先

  disclosure / 行点击失灵。

## [0.2.0] — 2026-05-31

### 新增
- **懒加载 attach**：启动和切换 host 时，发现的 tmux 会话先以**休眠占位行**出现（状态 = Idle，
  月亮图标），只有在**首次被选中**时才真正建立 `tmux -CC` 连接。会话很多时，启动 / 切 host 不再
  一次性拉起几十个连接，瞬间完成、占用更省。

### 变更
- **AsyncStream 流控**：连接时启用 tmux 暂停模式（`refresh-client -f pause-after=1`）。某个终端
  疯狂输出（如 `yes`、大日志）时，不再无上限堆积内存、也不再拖垮整个界面——tmux 会缓冲并暂停，
  积压以 `%extended-output` 流式送达，TFA 渲染后回 `continue` 恢复实时。对过旧、不支持流控的
  tmux 会**静默降级**（不报错）。

### 说明
- 休眠（尚未打开过）的终端没有活动连接，因此它的后台「活动指示点」要等首次打开后才会亮起——
  这是懒加载的固有取舍。
